STEP 01

Put the design online

~15 MIN

The design is currently just HTML files in this project. We'll deploy them to Vercel — it's free, takes 5 minutes, and gives you a URL like calendar-yourname.vercel.app.

  1. Sign up for Vercel with your GitHub account (or email).
  2. Export this project as a zip from this app's menu, unzip it on your computer.
  3. In Vercel, click "Add New… → Project", then "Import from local", and drag the unzipped folder in.
  4. When asked which file to serve, pick Wall Calendar v3.html — or rename it to index.html first so Vercel finds it automatically.
  5. Click Deploy. ~30 seconds later you'll get your URL.
WHY

You could also just open the HTML file directly on the laptop via file:///, but hosting it means: (a) the calendar proxy in step 2 can talk to it, (b) you can update the design from anywhere, (c) the laptop only needs Chrome and a URL.

STEP 02

Connect your Google Calendar

~25 MIN

We're going to use Google Calendar's private iCal feed — a secret URL that contains all your events. We don't use the full Google API because it requires OAuth, and OAuth on a kiosk display is a hassle. The iCal feed is simpler and updates every few hours, which is fine for a wall display.

2a · Grab your private iCal URL

  1. Open Google Calendar on a computer.
  2. Hover the calendar you want in the left sidebar → ⋮ → Settings and sharing.
  3. Scroll to "Integrate calendar".
  4. Copy the "Secret address in iCal format". It looks like https://calendar.google.com/calendar/ical/…/private-abc123/basic.ics.
HEADS UP

This URL is a password. Anyone who has it can see all your events. Don't paste it into the HTML, don't commit it to GitHub. It goes into the serverless function as an environment variable (next step), where only Vercel can read it.

2b · Add a tiny proxy function

Google's iCal endpoint doesn't allow browsers to fetch it directly (CORS). So we put a 30-line serverless function on Vercel that fetches it on the server side and hands it to the page as JSON.

In your project folder, create api/events.js:

// api/events.js — runs on Vercel as a serverless function.
// Reads your secret iCal URL from env, returns parsed events as JSON.
import ical from 'node-ical';

export default async function handler(req, res) {
  try {
    const data = await ical.async.fromURL(process.env.GCAL_ICS_URL);
    const events = Object.values(data)
      .filter((e) => e.type === 'VEVENT')
      .map((e) => ({
        id: e.uid,
        title: e.summary,
        loc: e.location || '',
        start: e.start.toISOString(),
        end: e.end.toISOString(),
      }));
    res.setHeader('Cache-Control', 's-maxage=300'); // cache 5 min
    res.status(200).json({ events });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
}

And a one-line package.json:

{ "dependencies": { "node-ical": "^0.18.0" } }

2c · Set the secret URL in Vercel

  1. Vercel project → Settings → Environment Variables.
  2. Add GCAL_ICS_URL = your secret iCal URL.
  3. Redeploy. Your function is now live at https://your-site.vercel.app/api/events — visit it to see today's events as JSON.

STEP 03

Prepare the Windows laptop

~15 MIN

3a · Rotate the display to portrait

  1. Plug the laptop into the TV via HDMI. Mount the TV vertically.
  2. Right-click the desktop → Display settings.
  3. Select the TV display → Display orientation → Portrait.
  4. Set the TV as the main display, and "Show only on" the TV if you want the laptop's screen off.

3b · Never sleep, never lock

  1. Power Options → set both "Plugged in" sliders to Never (turn off the display, put to sleep).
  2. Advanced power settings → "When I close the lid" → Do nothing (so you can fold the laptop behind the TV).
  3. Lock screen settings → screen timeout = Never.
  4. Windows Update → set active hours to a wide range so it doesn't reboot mid-day. Schedule restarts overnight if possible.

3c · Install Chrome

Standard Chrome install. We use it instead of Edge because the kiosk flags are well-documented.

STEP 04

Kiosk mode + autoboot

~10 MIN

4a · Make a launcher batch file

Create C:\WallCalendar\launch.bat:

@echo off
REM Wait a few seconds for network on boot
timeout /t 10 /nobreak > nul

REM Launch Chrome in kiosk mode pointing at your Vercel URL
start "" "C:\Program Files\Google\Chrome\Application\chrome.exe" ^
  --kiosk ^
  --noerrdialogs ^
  --disable-infobars ^
  --disable-session-crashed-bubble ^
  --disable-features=TranslateUI ^
  --no-first-run ^
  --start-fullscreen ^
  --incognito ^
  "https://your-site.vercel.app"

Replace the URL with your actual Vercel URL. Test it: double-click launch.bat — Chrome should pop up fullscreen with no chrome (no URL bar, no tabs).

4b · Auto-launch on Windows boot

  1. Press Win + R, type shell:startup, hit Enter.
  2. Right-click in the folder that opens → New → Shortcut → point at your launch.bat.
  3. Restart the laptop to test. It should auto-log-in to the desktop, wait 10 seconds, and launch the calendar fullscreen.

4c · Auto-login on boot

By default Windows asks for a password on startup. To skip it:

  1. Press Win + R, type netplwiz, hit Enter.
  2. Uncheck "Users must enter a user name and password to use this computer".
  3. Enter your account password to confirm.
SECURITY

This means anyone with physical access to the laptop is logged in. Fine for a wall display behind a TV; not fine for a laptop you carry around. Don't enable this on a laptop you also use for work or personal stuff.

4d · Hide the mouse cursor

Chrome's kiosk mode auto-hides the cursor after ~3 seconds of inactivity. If you want it gone entirely, install AutoHideMouseCursor (free, tiny, set delay to 1 second).

STEP 05

Mount + go

~30 MIN PHYSICAL
  1. Mount the TV portrait. A standard VESA mount usually rotates; or use a third-party portrait mount.
  2. Run the laptop's power cable to a wall outlet — ideally one not on a switched circuit.
  3. Plug the laptop in, plug the HDMI in. Fold the laptop closed.
  4. Power on. ~20 seconds later: the calendar.
PRO TIP

Put both the laptop and TV on a smart plug. If the calendar ever wedges (Chrome crashes, Windows updates do something weird), you can power-cycle the whole thing remotely from your phone.
